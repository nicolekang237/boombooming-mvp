## E. (2차 검증) 구조적으로 기능 구현이 불가능하거나 막혀 있는 지점

기계 검사 결과 깨진 링크 0건, 누락 이미지 0건 — 파일 참조 자체는 건강함. 대신 아래 구조 문제 확인.

### E-1. 🔴 카메라 녹화 영상이 게시 화면에 도달할 수 없는 구조
- upload_main.html:894 — 녹화 결과를 `sessionStorage.setItem('lastRecordingUrl', URL.createObjectURL(blob))`로 저장하지만, **이 키를 읽는 곳이 프로젝트 전체에 없음**.
- 더 근본적으로, blob URL은 페이지를 이탈하면 소멸하므로 `location.href='./upload_details.html'`로 넘어간 순간 녹화 영상 데이터는 사용 불가. 현재 구조에서 카메라로 찍은 영상은 **썸네일 1장만 전달되고 영상 본체는 버려짐**.
- 개발 시 IndexedDB 저장, 즉시 서버 업로드, 또는 네이티브 브릿지 중 하나로 파일 전달 구조를 새로 설계해야 함 (프로토타입 수정으로 해결 안 되는 아키텍처 항목).

### E-2. 🔴 리나 강 채팅방(수익화 핵심)에 스크립트가 0개
- chatroom_rina.html — 결제 유도(rina_payment)와 AI 매칭 동의(signup)의 중심축인데 `<script>`가 아예 없음. 메시지 전송도, AI 응답 시뮬레이션도 없음. 데모 시연에서 "리나와 대화" 시나리오 자체가 재현 불가.

### E-3. 🟠 홈 스와이프 데모의 함정 3가지 (home.html)
1. **`bbm_myVideoCount` 기본값 0** (L1204) → 오른쪽 스와이프(인맥신청)가 항상 "최소 1개의 영상이 있어야…" 토스트로 차단됨. 콘솔에서 값을 넣어야만 매칭 플로 시연 가능 — 시연자가 모르면 데모가 고장난 것처럼 보임.
2. **카드 6장을 다 넘기면 빈 검은 화면** — 영상 소진 상태(empty state)도, 새 카드 리필도 없음.
3. **CARD_USERS 배열이 카드 DOM 순서에 하드코딩** (L1032) — 카드 추가/삭제/순서 변경 시 영상과 인물이 어긋남 (주석에도 경고 있음).

### E-4. 🟠 선택값이 결과에 반영되지 않는 데드엔드 데이터
- `uploadAspectRatio` (upload_details.html:414) — 화면비(9:16/1:1/4:5/16:9)를 저장만 하고 **읽는 곳이 없음**. 피드/상세 어디에도 화면비가 반영되는 경로 없음.

### E-5. 🟡 외부 CDN 의존 — 오프라인 시연 불가
- home.html:66 Hammer.js(스와이프 전체가 의존), upload_main.html:12 DeepAR SDK. 네트워크 없는 환경에선 홈 스와이프와 카메라 효과가 통째로 죽음. 로컬 번들 권장.

### E-6. 🟡 공통 레이아웃이 페이지마다 복제된 구조
- 헤더/푸터가 43개 파일에 각각 복사돼 있고(파일 내 주석에도 "인클루드로 분리해야 함" 명시), 탭 명칭이 페이지마다 불일치: **"메시지" vs "대화", "만들기" vs "업로드"**. 활성 탭 표시도 주석 처리 토글 방식. 개발 이관 시 컴포넌트화 전제 필요.
- 로그인 전 화면(signup, find_id, find_password, reset_password, terms, privacy)에 **로그인 후 전용 하단 네비게이션이 그대로 노출** — signup 헤더에는 chat_directory로 가는 버튼까지 있음. 인증 전/후 레이아웃 분리 필요.

### E-7. 🟡 기타 2차 발견
- 홈 매칭 성공 팝업의 "메시지 보내기"가 매칭된 상대가 아닌 범용 chatroom.html로 이동, 입력한 첫 메시지도 버려짐 (home.html:1111).
- 매칭 성사 데이터(`bbm_connections`)를 채팅 목록(chat.html)이 읽지 않음 — 매칭돼도 채팅 목록에 새 방이 생기지 않아 플로가 끊김.

1. **계정 성립 (🔴)**: 회원가입 비밀번호 + 필수약관 동의 + signup 스크립트 / 인증코드 입력 화면 / 아이디·비번 찾기 동작 + 결과 화면
2. **신뢰·안전 (🔴)**: 차단 목록 화면 + settings 진입 행 / 채팅방 내 차단·신고·나가기 시트 / 홈 신고 버튼 → report.html 연결
3. **핵심 루프 (🔴)**: 채팅 전송 동작 / 내 영상 관리(mychannel 연결 + 수정·삭제) / video_detail 더보기 시트 / 15초~3분 검증 + 녹화 오디오
4. **수익화 (🔴~🟠)**: 결제 내역 / 구독 해지 / 영수증
5. **완성도 (🟠~🟡)**: 북마크 보관함 / 임시저장·이탈 확인 / 설정 하위 화면 / 문의 내역·공지사항 / 좋아요·즐겨찾기 토글 / 인맥 버튼 연결
